# Engineering Workflows

Codex-adapted engineering workflow skills for diagnosis, TDD, planning interviews, architecture
review, and codebase orientation.

These skills are adapted from Matt Pocock's MIT-licensed
[`mattpocock/skills`](https://github.com/mattpocock/skills) repository. Each adapted skill includes
Agent Skills frontmatter with `license: MIT` plus metadata for the original author, source path, and
adaptation flag. The MIT license notice from the source repository is preserved in
[LICENSE](./LICENSE).

## Skills

- `engineering-workflows:diagnose`: Run a disciplined diagnosis loop for bugs, flaky behavior, and
  performance regressions.
- `engineering-workflows:tdd`: Build features or fixes with a red-green-refactor loop.
- `engineering-workflows:grill-me`: Stress-test a plan or design through a focused interview.
- `engineering-workflows:grill-with-docs`: Stress-test a plan while preserving durable domain terms
  and ADRs.
- `engineering-workflows:zoom-out`: Map an unfamiliar code area at a higher level of abstraction.
- `engineering-workflows:improve-codebase-architecture`: Find module deepening and architecture
  improvement opportunities.
